# Established: fluent in the tool's own capacity model

Through the design/grilling session, the learner is already fluent in the
Heatmap Modeler's model: unified Load Intensity, Subtractive vs Additive modes,
Base Capacity + Reducers, Consumers (Recurring/Spanning), Dimensions/faceting,
and rollup-by-summation. Future lessons should build on this vocabulary (use
`CONTEXT.md` terms) rather than re-teaching single-Subject modelling.

Implications: the productive frontier is *multi-source* capacity (composition,
constraints), *effective* capacity (shrinkage/focus factor), and utilisation
target bands — not the basics.
