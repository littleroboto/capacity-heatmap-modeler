Capacity Modeling · Reference

Glossary

The shared vocabulary of the Capacity Heatmap Modeler. Every lesson uses these words precisely — this page is the source of truth. Mirrors the project CONTEXT.md.

Core model

Subject
The entity a single heatmap models — determined by the current faceting level, not a fixed type. Facet by person → each Subject is a person; don't facet → the Subject is the whole aggregate. "Team" and "business" are just aggregates that sum across un-faceted Dimensions.
Avoid: entity, actor, resource
Cell
The atomic unit: one Subject on one day. Its colour encodes Load Intensity. Days are the fixed grain; coarser views are Roll-up Views.
Avoid: box, square
Roll-up View
A weekly or monthly aggregate rendered on the daily Cells (intensity aggregated over the bucket). The underlying grain is always daily.
Avoid: zoom, summary
Load Intensity 0 → 1+
The single normalised value every Cell is painted with, on one blue→red ramp. 0 = empty (coolest), 1.0 = the full-red Reference, >1.0 = beyond it (deepest red).
Avoid: heat, utilisation (utilisation is one interpretation of intensity)
Reference
The quantity that maps to Load Intensity 1.0 (full red) for a Cell. What sets the Reference is the only thing that differs between the two Modes.
Avoid: max, ceiling, threshold (each is a specific kind of Reference)

Modes

Subtractive mode
The Subject has a declared Capacity. A Cell starts blue and moves toward red as Consumers use it up. Reference = Effective Capacity, so Load Intensity = consumed ÷ effective capacity; >1.0 is real, meaningful overload.
Avoid: capacity mode, top-down
Additive mode
The Subject has no declared Capacity. A Cell starts empty and heats up as Consumers layer in. Reference = a Normalisation Max (relative), so Load Intensity is relative "hotness" with no true overload.
Avoid: bottom-up, layering mode

Capacity, reducers, consumers

Capacity
A declared block of available "room" for a Subject, in one Declared Unit. One heatmap has exactly one Capacity in one unit. Subtractive mode only.
Avoid: supply, budget
Declared Unit
The single unit a heatmap's Capacity and Consumers are measured in (minutes, covers, GBP, labs, story-points). Free-text per Scenario; the tool does not convert between units.
Avoid: nominal scale, measure
Base Capacity
The declared capacity per day before Reducers, in the Declared Unit. One amount with an optional weekday pattern (e.g. weekdays 450, weekends 0). Default flat.
Avoid: raw capacity, gross capacity
Effective Capacity
Base Capacity × product of all active Reducer multipliers for a Cell. This is the Reference in Subtractive mode.
Avoid: net capacity, available capacity
Reducer 0 → 1
Something that lowers a Subject's Capacity over a time span, as a multiplier in [0,1] (holiday = 0; reduced staffing = e.g. 0.6). Reducers stack multiplicatively. Subtractive mode only.
Avoid: blocker, deduction
Blackout
A day whose Effective Capacity is 0 (e.g. a holiday). Rendered neutral grey, not red. Consumer load landing on a Blackout is kept grey but flagged with a warning marker.
Avoid: closed, off-day
Consumer
Something that generates load over time (task, meeting, campaign, programme). Exists in both Modes. A day's total load is the sum of all active Consumers' amounts ("layering" = summation).
Avoid: demand, activity, task (these are kinds of a Consumer)
Recurring Consumer
Applies a per-active-day amount on a recurrence (every weekday, specific weekdays, a cadence), optionally bounded by a date window. For standing duties and BAU.
Avoid: repeating task
Spanning Consumer
Applies over a single dated range with a Load Shape describing how its amount varies across the range. For campaigns, programmes, projects.
Avoid: event, campaign (those are sources of one)
Load Shape
How a Spanning Consumer's amount is distributed across its range — flat, a ramp-up/down, or named Phases.
Avoid: curve, profile
Phase
A named sub-span of a Spanning Consumer's range carrying its own amount (e.g. prep low, then live high).
Avoid: stage, period

Composition (multi-source capacity)

Track
One capacity source contributing to a Subject when its capacity comes from more than one place (a person, a CI pipeline, a department). Tagged like any Capacity entry.
Avoid: stream, lane, channel
Composition
The per-Subject rule for combining Tracks into Effective Capacity: sum (Tracks fungible/parallel — capacities add) or min (Tracks coupled/serial — the weakest caps the whole). Default sum.
Avoid: aggregation, combine
Constraint
Under min composition, the Track with the least capacity — the one that caps output (the bottleneck). Improving a non-Constraint Track does not raise Effective Capacity.
Avoid: bottleneck (gloss only), limiter

Scenario, output & faceting

Scenario
One saved, self-contained model — its Subject(s), Capacity, Reducers, Consumers, and config — that renders to a one-page visual.
Avoid: plan, case, view
Normalisation Max
In Additive mode, the load value that maps to full red. May be the hottest Cell in the visual or a value set by the user.
Avoid: peak, cap
Dimension
A config-declared attribute with a fixed set of allowed values that every Capacity and Consumer can be tagged with (country, brand, channel). Choosing one to facet by splits a visual into small multiples.
Avoid: field, axis, tag (each a partial view of a Dimension)
Small multiples
The set of heatmaps produced by faceting a visual along one Dimension's values.
Avoid: panels, grid
