Capacity composition

How to combine capacity from more than one source. Keep this next to you while modelling.

The two composition rules

Sum — parallel / fungible

C = c₁ + c₂ + … + cₙ

Sources do interchangeable work; any unit can absorb the load. More units → more capacity.

3 cross-trained agents; identical machines; bench of developers.

Min — coupled / serial

C = min(c₁, c₂, …, cₙ)

Sources must work together to make output; the weakest link caps the whole. Adding to a non-constraint does nothing.

1 dev + 1 CI pipeline; cooks + one fryer; a specialist hand-off.

Test to tell them apart: ask “if I add one more unit of this source, does total output go up?” Yes for every source → sum. Only for one source → that source is the constraint; use min.

Utilisation bands

Utilisation = load ÷ effective capacity. Higher is not better.

  • < ~60% — slack / over-provisioned (a cost).
  • ~70–85% — healthy, sustainable band. Aim here.1
  • > ~90% — fragile; queues and delays grow fast; burnout / breakage risk.
Effective capacity is smaller than raw capacity. Subtract non-productive time (leave, meetings, admin — often 5–8%+) and apply a focus factor before comparing to demand.2
