# Teaching Notes

## Learner preferences
- Likes **active decisions** — grilling / multiple-choice with a recommendation
  first, one thing at a time. (Evidenced across the design session.)
- Terse. Recommendations up front, then rationale.
- Wants everything grounded in the tool being built (CONTEXT.md + ADRs).

## Prior knowledge (as of setup)
Already fluent in the tool's own model from the design session: unified
intensity, Subtractive vs Additive modes, Reducers vs Consumers, Dimensions /
faceting, rollup = summation. Don't re-teach these — build on them.

## Teaching approach
- Lessons short, one tangible win, interactive feedback loop.
- Tie each lesson to a concrete modelling decision in the tool.
- Flag when a lesson exposes a real design implication (candidate ADR / gap).
